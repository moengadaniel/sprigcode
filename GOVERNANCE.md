# Sprigcode Governance

Sprigcode is maintainer-led.

- Lead maintainer: `moengadaniel`
- GitHub: `moengadaniel`

## What the lead maintainer decides

- project direction
- public API design
- architecture
- release approval
- maintainer delegation
- governance changes

## Why this model fits the project

Sprigcode is trying to establish a trusted primitive, not a broad platform.
That requires tight control over public claims and failure semantics.

## Contribution model

Contributors are welcome through issues, discussions, RFCs, and pull requests.
Major design changes should come with an RFC and explicit reasoning about:

- the problem
- the failure model
- the test plan
- the scope cost

## Limitation

This model optimizes for coherence and discipline. It does not optimize for
maximum feature throughput.
