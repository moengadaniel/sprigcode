# Roadmap

## v0.1 line

The goal is not breadth. The goal is credibility.

Priority order:

1. make the TypeScript and TSX slice excellent
2. strengthen fail-closed behavior
3. make demos and docs immediately convincing
4. keep package and CLI ergonomics clear for public use

## Near-term follow-up

- add harder TypeScript fixture coverage
- strengthen output and package-publishing smoke checks
- document integration patterns for tools and agents

## Not on the short path

- broad language support
- broad syntax support
- recipe systems layered over an unproven primitive
- “AI platform” features

## Limitation

Sprigcode only becomes more ambitious after the current slice is obviously
useful and obviously safe.
